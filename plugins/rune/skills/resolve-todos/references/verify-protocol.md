# Verify Protocol — Phase 3

Verifier agent prompts, verdict taxonomy, confidence thresholds, and sanitization.

## Verdict Taxonomy (7 verdicts)

| Verdict | When | Confidence Floor |
|---------|------|------------------|
| `VALID` | TODO is real, actionable, and needs fixing | >= 0.70 |
| `FALSE_POSITIVE` | TODO is hallucinated or already addressed differently | >= 0.85 |
| `ALREADY_FIXED` | Code already contains the fix | >= 0.80 |
| `NEEDS_CLARIFICATION` | Insufficient information to verify | < 0.70 |
| `PARTIAL` | Multi-part TODO only partially resolved | >= 0.70 |
| `DUPLICATE` | Same file+line as another TODO in batch | >= 0.90 |
| `DEFERRED` | Valid but too risky to fix now | >= 0.70 |

## Verifier Agent Prompt Template

```markdown
You are a TODO verification specialist. Your job is to determine if each TODO
is VALID (real issue that needs fixing) or FALSE_POSITIVE (already fixed, not applicable,
or hallucinated by a previous review).

## ANCHOR — TRUTHBINDING
You MUST verify against ACTUAL code. Do NOT trust the TODO description blindly.
The TODO may have been generated by an AI that hallucinated. CHECK THE CODE.

## File: {file}

## Context Summary:
<context nonce="{nonce}">
{contextSummary}
</context>

NOTE: {contextSummary} = aggregated output from Phase 2 context agents, capped at 2000 chars.
{nonce} = UUID-v4 generated per verifier invocation. Used to scope untrusted content.
{todosList} = sanitized TODO descriptions (post-sanitizeTodoBody), placed INSIDE the nonce boundary.

## TODOs to Verify:
<todos nonce="{nonce}">
{todosList}
</todos>

## Verification Checklist (per TODO)
0. **Locate actual code** — grep-first, don't trust line numbers (drift detection)
1. Does the file actually contain the code referenced by the TODO?
2. Has the issue already been fixed in current code?
3. Is the TODO recommendation technically correct?
4. Would the suggested fix introduce regressions?
5. Does the TODO match the file's actual behavior (not assumed behavior)?

## Output
Write to: tmp/resolve-todos-{timestamp}/verdicts/{file}.json
Format:
{
  "file": "{file}",
  "verdicts": [
    {
      "todo_id": "001",
      "verdict": "VALID|FALSE_POSITIVE|ALREADY_FIXED|NEEDS_CLARIFICATION|PARTIAL|DUPLICATE|DEFERRED",
      "confidence": 0.0-1.0,
      "evidence": "Line 42: the function still uses raw SQL without parameterization",
      "risk_if_fixed": "low|medium|high",
      "suggested_approach": "Use parameterized query with $1 placeholder"
    }
  ]
}

## RE-ANCHOR — TRUTHBINDING
Base ALL verdicts on code you actually read. If you cannot find the referenced code,
mark as FALSE_POSITIVE with evidence "referenced code not found at stated location".
```

## Line Number Drift Detection

```javascript
// Verifier Step 0: Locate actual code
// Don't trust line numbers from the TODO — they may have drifted
const fileContent = Read(file)
for (const todo of todos) {
  const snippet = extractSnippet(todo.description)
  const actualLine = fileContent.split('\n').findIndex(l => l.includes(snippet))
  if (actualLine === -1) {
    todo.verdict = "FALSE_POSITIVE"
    todo.evidence = "referenced code not found at stated location (line drift or deletion)"
  }
}
```

## TODO Body Sanitization (Injection Defense)

```javascript
function sanitizeTodoBody(body) {
  // Pass 1: Strip control patterns
  let clean = body
    .replace(/<!--[\s\S]*?-->/g, '')              // HTML comments
    .replace(/```[\s\S]*?```/g, '[code]')          // Code fences
    .replace(/<[^>]+>/g, '')                        // Inline HTML
    .replace(/\u200b|\u200c|\u200d|\ufeff/g, '')    // Zero-width chars
    .replace(/[\u200B-\u200F\uFEFF\uFE00-\uFE0F]/g, '')  // Zero-width + variation selectors
    .replace(/\uDB40[\uDC00-\uDC7F]/g, '')          // Tag block characters (U+E0000-E007F)
    .replace(/[\u202A-\u202E\u2066-\u2069]/g, '')   // Unicode directional overrides (CVE-2021-42574)
    .replace(/<\/context>/gi, '[/context]')          // SEC-008: prevent nonce boundary escape

  // Pass 2: Detect SUSPECT patterns (SEC-005: extended injection vectors)
  const SUSPECT_PATTERNS = [
    /ignore\s+(previous|above|all)\s+instructions/i,
    /you\s+are\s+(now|a)\s+/i,
    /system\s*:\s*/i,
    /<<<\s*NONCE/i,
    /act\s+as\s+/i,                                  // Indirect role hijacking
    /pretend\s+(you\s+are|to\s+be)\s+/i,             // Indirect role hijacking
    /in\s+the\s+role\s+of\s+/i,                      // Indirect role hijacking
    /ANCHOR\s*[—\-]+\s*TRUTHBINDING/i,               // ANCHOR spoofing
    /RE-ANCHOR/i,                                     // RE-ANCHOR spoofing
    /atob\s*\(/i,                                     // Base64 decoding attempt
    /Buffer\.from\s*\(/i,                             // Node.js base64 decoding
    /\b(disregard|forget)\s+(all|previous|above)\s+(instructions|rules)/i, // Instruction override variants
    /\bfrom\s+now\s+on\b/i,                           // Behavioral override
    /\bassistant\s*:/i,                                // Role injection (ChatML)
    /\bhuman\s*:/i,                                    // Role injection (ChatML)
  ]
  const matchedPattern = SUSPECT_PATTERNS.find(p => p.test(clean))
  const isSuspect = !!matchedPattern
  if (isSuspect) {
    clean = `[SUSPECT CONTENT REDACTED — manual review required]`
  }

  return { clean, isSuspect, matchedPattern: matchedPattern?.source ?? null }
}
```

## Confidence Thresholds

| Verdict | Confidence Floor | Rationale |
|---------|------------------|-----------|
| `VALID` | >= 0.70 | Standard Inner Flame threshold |
| `ALREADY_FIXED` | >= 0.80 | Must be certain code was changed |
| `FALSE_POSITIVE` | >= 0.85 | High bar — wrong dismissal is costly |
| `NEEDS_CLARIFICATION` | < 0.70 | Safe default for uncertain verdicts |
| `PARTIAL` | >= 0.70 | Some resolution found |
| `DUPLICATE` | >= 0.90 | Must match exactly |
| `DEFERRED` | >= 0.70 | Valid but deferred for risk reasons |
