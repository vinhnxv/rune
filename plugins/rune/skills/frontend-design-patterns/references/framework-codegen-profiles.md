# Framework Codegen Profiles — Post-Processing Transformation Rules

Post-processing profiles that transform generic figma-to-react MCP output into framework-native code. Applied after token resolution as the final codegen step.

## Profile Selection

The active profile is determined by `design-system-profile.yaml` (generated by `discoverDesignSystem()`):

```
figma-to-react MCP output → Token Resolver → Codegen Profile → Final code
(generic React + Tailwind)   (semantic swap)   (framework transform)  (native code)
```

| Detected Framework | Profile | Token Map |
|-------------------|---------|-----------|
| `shadcn` | shadcn | [shadcn-token-map.yaml](profiles/shadcn-token-map.yaml) |
| `untitled-ui` | UntitledUI | [untitled-ui-token-map.yaml](profiles/untitled-ui-token-map.yaml) |
| `generic` / `unknown` | Generic | (Tailwind palette fallback) |

## Transformation Rules Matrix

| Dimension | shadcn | UntitledUI | Generic |
|-----------|--------|------------|---------|
| **Import aliasing** | `@/components/ui/*` | `@/components/{tier}/*` | `@/components/*` |
| **Variant system** | `cva()` from class-variance-authority | `sortCx()` with CSS Module classes | Props + conditional `clsx()` |
| **Class merging** | `cn()` = `twMerge(clsx())` | `cx()` = `clsx()` (no twMerge needed) | `clsx()` |
| **Polymorphism** | `Slot` from `@radix-ui/react-slot` + `asChild` | React Aria Components composition | Native HTML + `as` prop |
| **Data attributes** | `data-slot` on component parts | `data-*` responsive attributes | None (class-based) |
| **Icon pattern** | `lucide-react` named exports | `@untitledui/icons` named exports | Inline SVG or generic icon lib |
| **File location** | `src/components/ui/{name}.tsx` | `src/components/{tier}/{Name}/` | `src/components/{Name}/` |
| **Dark mode** | `.dark` class on `<html>` + oklch CSS vars | `[data-theme="dark"]` + RGB CSS vars | `prefers-color-scheme` media query |
| **Color space** | oklch (Delta-E 2000 matching) | RGB hex (Delta-E 2000 for oklch tokens) | hex/hsl (RGB Euclidean fallback) |

## shadcn Profile

### Import Transform

```
BEFORE (figma-to-react output):
  import React from "react"

AFTER (shadcn transform):
  import * as React from "react"
  import { cn } from "@/lib/utils"
  import { cva, type VariantProps } from "class-variance-authority"
  import { Slot } from "@radix-ui/react-slot"
```

**Rules:**
- Replace `import React` with `import * as React` (shadcn convention)
- Add `cn` import when any className manipulation is present
- Add `cva` import when 2+ visual variants are detected
- Add `Slot` import when `asChild` prop pattern is applicable

### Variant Transform

When figma-to-react detects multiple Figma variants of the same component, transform conditional classes into `cva()`:

```
BEFORE:
  const classes = isDestructive
    ? "bg-red-500 text-white"
    : "bg-blue-500 text-white"

AFTER:
  const buttonVariants = cva(
    "inline-flex items-center justify-center text-sm font-medium transition-colors",
    {
      variants: {
        variant: {
          default: "bg-primary text-primary-foreground shadow hover:bg-primary/90",
          destructive: "bg-destructive text-destructive-foreground shadow-sm hover:bg-destructive/90",
        },
      },
      defaultVariants: {
        variant: "default",
      },
    }
  )
```

### Class Merge Transform

All `className` concatenation must use `cn()`:

```
BEFORE:
  className={`px-4 py-2 ${className}`}

AFTER:
  className={cn("px-4 py-2", className)}
```

### Polymorphism Transform

Components that render as multiple element types get `asChild` + `Slot`:

```
BEFORE:
  const Comp = as || "button"
  return <Comp className={classes}>{children}</Comp>

AFTER:
  const Comp = asChild ? Slot : "button"
  return <Comp className={cn(buttonVariants({ variant, size, className }))} ref={ref}>{children}</Comp>
```

### Component Props Transform

```typescript
// shadcn prop pattern
export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {
  asChild?: boolean
}

const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant, size, asChild = false, ...props }, ref) => {
    // ...
  }
)
Button.displayName = "Button"
```

### File Location

```
src/components/ui/{name}.tsx          ← single-file component
src/components/ui/{name}/index.tsx    ← only if compound (Dialog, Dropdown)
```

### Dark Mode

```css
/* oklch custom properties in globals.css */
:root {
  --primary: oklch(0.205 0 0);
  --primary-foreground: oklch(0.985 0 0);
}
.dark {
  --primary: oklch(0.922 0 0);
  --primary-foreground: oklch(0.205 0 0);
}
```

Transform rule: Never emit Tailwind `dark:` variants. All dark mode handled through CSS variable switching.

## UntitledUI Profile

### Import Transform

```
BEFORE (figma-to-react output):
  import React from "react"

AFTER (UntitledUI transform):
  import React from "react"
  import { cx } from "@/lib/cx"
  import styles from "./{Name}.module.css"
```

**Rules:**
- Add `cx` (lightweight clsx wrapper) when class merging is needed
- Add CSS Module import for component styles
- Add React Aria imports for interactive components

### Variant Transform

UntitledUI uses CSS Module class-based variants with a `sortCx()` helper:

```
BEFORE:
  const classes = isDestructive
    ? "bg-red-500 text-white"
    : "bg-purple-600 text-white"

AFTER:
  // In {Name}.module.css:
  .btn { /* base styles using CSS vars */ }
  .btn--primary { background-color: var(--bg-brand-primary); color: var(--fg-white); }
  .btn--destructive { background-color: #D92D20; color: var(--fg-white); }
  .btn--sm { height: 36px; padding: 0 var(--spacing-sm); }
  .btn--md { height: 40px; padding: 0 var(--spacing-md); }

  // In component:
  className={cx(
    styles.btn,
    styles[`btn--${variant}`],
    styles[`btn--${size}`],
    className
  )}
```

### Class Merge Transform

UntitledUI uses `cx()` (not `cn()`/`twMerge`) since CSS Modules avoid class conflicts:

```
BEFORE:
  className={`px-4 py-2 ${className}`}

AFTER:
  className={cx(styles.container, className)}
```

### Accessibility Transform

Interactive components get React Aria hooks or components:

```
BEFORE:
  <button onClick={onPress} disabled={isDisabled}>

AFTER (hooks approach):
  import { useButton } from "@react-aria/button"

  const ref = useRef(null)
  const { buttonProps } = useButton({ onPress, isDisabled }, ref)
  return <button {...buttonProps} ref={ref} className={cx(styles.btn)}>

AFTER (components approach):
  import { Button } from "react-aria-components"

  return <Button onPress={onPress} isDisabled={isDisabled} className={cx(styles.btn)}>
```

### Component Props Transform

```typescript
// UntitledUI prop pattern
interface ButtonProps {
  variant?: "primary" | "secondary-color" | "destructive"
  size?: "sm" | "md" | "lg"
  className?: string
  children: React.ReactNode
  onPress?: () => void
  isDisabled?: boolean
}

function Button({ variant = "primary", size = "md", className, children, ...props }: ButtonProps) {
  return (
    <button
      className={cx(styles.btn, styles[`btn--${variant}`], styles[`btn--${size}`], className)}
      {...props}
    >
      {children}
    </button>
  )
}
```

### File Location — 3-Tier Directory

```
src/components/base/{Name}/           ← Tier 1: Atoms (Button, Input, Badge)
  ├── {Name}.tsx
  ├── {Name}.module.css
  └── index.ts
src/components/application/{Name}/    ← Tier 2: Molecules/organisms (Modal, DataTable)
src/components/marketing/{Name}/      ← Tier 3: Page sections (Hero, PricingCard)
```

**Tier assignment rules:**
- No business logic, no state management → `base/`
- Reads stores/context, app-specific → `application/`
- Static/CMS-driven page sections → `marketing/`

### Dark Mode

```css
/* data-theme attribute on <html> */
:root { --fg-primary: #101828; --bg-primary: #ffffff; }
[data-theme="dark"] { --fg-primary: #f9fafb; --bg-primary: #0c111d; }
```

Transform rule: Never emit Tailwind `dark:` variants or `.dark` class. Use `[data-theme="dark"]` for all dark mode selectors.

### Icon Transform

```
BEFORE:
  <svg>...</svg>

AFTER:
  import { ArrowRight } from "@untitledui/icons"
  <ArrowRight className={styles.icon} />
```

## Generic Profile

### Import Transform

```
BEFORE (figma-to-react output):
  import React from "react"

AFTER (generic transform):
  import React from "react"
  import clsx from "clsx"
```

**Rules:**
- Add `clsx` for class merging (no twMerge dependency)
- No CSS Module assumption (inline Tailwind or utility classes)
- No third-party primitive library assumed

### Variant Transform

Generic profile uses props + conditional classes:

```
BEFORE:
  const classes = isDestructive
    ? "bg-red-500 text-white"
    : "bg-blue-500 text-white"

AFTER:
  const variantClasses = {
    primary: "bg-blue-500 text-white hover:bg-blue-600",
    secondary: "bg-gray-100 text-gray-900 hover:bg-gray-200",
    destructive: "bg-red-500 text-white hover:bg-red-600",
  }

  className={clsx("inline-flex items-center px-4 py-2 rounded", variantClasses[variant], className)}
```

### Class Merge Transform

```
BEFORE:
  className={`px-4 py-2 ${className}`}

AFTER:
  className={clsx("px-4 py-2", className)}
```

### Polymorphism Transform

Generic profile uses `as` prop (no Slot dependency):

```
BEFORE:
  <button className={classes}>{children}</button>

AFTER:
  const Component = as || "button"
  return <Component className={clsx(baseClasses, className)} {...props}>{children}</Component>
```

### Component Props Transform

```typescript
// Generic prop pattern
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: "primary" | "secondary" | "destructive"
  size?: "sm" | "md" | "lg"
  as?: React.ElementType
}

function Button({ variant = "primary", size = "md", as, className, children, ...props }: ButtonProps) {
  const Component = as || "button"
  return (
    <Component
      className={clsx(
        "inline-flex items-center justify-center font-medium transition-colors",
        variantClasses[variant],
        sizeClasses[size],
        className
      )}
      {...props}
    >
      {children}
    </Component>
  )
}
```

### File Location

```
src/components/{Name}/
  ├── {Name}.tsx
  ├── {Name}.test.tsx
  └── index.ts
```

No tier structure. Flat component directory.

### Dark Mode

```css
/* prefers-color-scheme media query */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #111827;
    --color-text: #f9fafb;
  }
}
```

Transform rule: Use `prefers-color-scheme` media query. Also accept Tailwind `dark:` variants as an alternative when the project uses Tailwind's dark mode class strategy.

## Post-Processing Pipeline

The codegen profile runs as the final step in the design-sync pipeline:

```
1. figma_to_react() → raw React + Tailwind output
2. Token Resolver   → semantic token replacement (framework-specific)
3. Codegen Profile  → structural transforms:
   a. Import rewriting (add framework deps, remove unused)
   b. Variant extraction (conditional classes → variant system)
   c. Class merge wrapping (string concat → cn/cx/clsx)
   d. Polymorphism injection (asChild/React Aria/as prop)
   e. File structure (single-file vs directory, tier placement)
   f. Dark mode normalization (remove hardcoded, ensure CSS var usage)
   g. Icon replacement (inline SVG → icon library imports)
4. Write final files to disk
```

### Profile Override

Projects can override the detected profile in `talisman.yml`:

```yaml
design_sync:
  codegen_profile: "shadcn"    # Force shadcn profile regardless of detection
```

Or in `design-system-profile.yaml`:

```yaml
framework: "shadcn"
codegen_profile_override: null  # null = auto-detect, or force a profile name
```
