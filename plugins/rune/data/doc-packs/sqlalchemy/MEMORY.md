# SQLAlchemy Doc Pack

## Etched — SQLAlchemy: 2.0 Query Style (2026-03-01)

**Source**: `doc-pack:sqlalchemy@1.0.0`
**Category**: pattern

### Modern Query Patterns (2.0 Style)
- `select(User).where(User.id == 1)` replaces `session.query(User).filter_by(id=1)`
- `session.execute(stmt)` returns `Result` — use `.scalars()` for ORM objects
- `.scalars().all()` for list, `.scalars().first()` for optional single, `.scalars().one()` for exactly one
- `insert(User).values(name="Alice")` and `update(User).where(...).values(...)` for DML

### Relationship Patterns
- `Mapped[list["Post"]]` replaces `relationship("Post", backref="author")`
- `mapped_column()` replaces `Column()` — type-safe column definitions
- `relationship(back_populates="author")` preferred over `backref` (explicit is better)
- `selectinload(User.posts)` for eager loading — avoids N+1 in async contexts

## Etched — SQLAlchemy: Async Session Patterns (2026-03-01)

**Source**: `doc-pack:sqlalchemy@1.0.0`
**Category**: pattern

### Async Setup
```python
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession

engine = create_async_engine("postgresql+asyncpg://...", pool_size=5, max_overflow=10)
async_session = async_sessionmaker(engine, expire_on_commit=False)
```

### Async Gotchas
- `expire_on_commit=False` required for async — prevents lazy load after commit
- `await session.refresh(obj)` to reload after commit if needed
- Lazy loading FAILS in async — always use eager loading (`selectinload`, `joinedload`)
- `session.run_sync()` for sync-only operations inside async context (escape hatch)
- `async with session.begin():` for transaction scope — auto-commits or rolls back

### Connection Pooling
- `pool_size=5` default — increase for high-concurrency apps
- `pool_recycle=3600` to prevent stale connections (especially with MySQL)
- `pool_pre_ping=True` for connection health checks before use
- Use `NullPool` for serverless (Lambda) — no persistent connections

## Etched — SQLAlchemy: Migration with Alembic (2026-03-01)

**Source**: `doc-pack:sqlalchemy@1.0.0`
**Category**: pattern

### Alembic Patterns
- `alembic revision --autogenerate -m "add users table"` — generates migration from model diff
- Always review autogenerated migrations — they miss: data migrations, index names, enum changes
- `alembic upgrade head` to apply, `alembic downgrade -1` to revert one step
- `alembic stamp head` to mark current DB as up-to-date without running migrations

### Migration Gotchas
- Autogenerate does NOT detect: column renames (shows as drop+add), table renames, data changes
- Always add `nullable=False` with a `server_default` — avoid breaking existing rows
- Use `batch_alter_table()` for SQLite (no ALTER TABLE support for most operations)
- Test migrations: `upgrade` then `downgrade` then `upgrade` again — must be idempotent
- Keep `env.py` target_metadata pointing to your Base.metadata — stale reference = empty migrations
