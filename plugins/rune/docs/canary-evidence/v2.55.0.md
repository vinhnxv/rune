# Canary Evidence Snapshot — v2.55.0

This snapshot records the evidence-gate verdict that unblocked the canary default flip for
`arc.state_file.code_enforced_writes` in v2.55.0. It is the artifact referenced by the
v2.55.0 CHANGELOG entry and by plan AC-4.

## How this file is populated

Before merging v2.55.0:

1. Run the gate script against the production arc integrity log:
   ```bash
   bash plugins/rune/scripts/observability/arc-state-health.sh --canary-gate
   ```
2. Expected verdict: `{ "gate_status": "PASS", "recommendation": "Ready for Phase B default flip" }`.
3. Paste the full JSON output into the "Gate output" section below.
4. Fill in the frontmatter placeholders (`gate_run_at`, `operator`, `git_sha`).
5. Commit this file with the same PR that flips the default.

## Evidence criteria (AC-4)

Gate thresholds from plan AC-4 must all be satisfied:

- (a) `verified_count >= 500` across fleet
- (b) `recovered_count + hydrated_count >= 10`
- (c) `corrupted_write_count == 0`
- (d) `layer2_mismatch_count == 0`
- (e) `deletion_deferred_vs_spurious_ratio > 1.0`
